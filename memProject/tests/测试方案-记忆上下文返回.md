## 测试用例

### 前置条件

测试前确保 `t_memory` 表中存在至少 5 条不同 `memory_type` 的记忆：

| memory_type | content 示例 |
|-------------|-------------|
| preference | "每天早上一杯美式咖啡" |
| task | "前端页面还剩 3 个接口没对接" |
| fact | "工作在张江科技园" |
| progress | "用户登录模块开发完成" |
| decision | "用 RocketMQ 解耦订单和库存服务" |

**API 写入（Mock 模式）：**
```json
{"user_id":"u_context","messages":[{"role":"user","content":"我喜欢喝咖啡"}]}
```

**数据库直接写入：**
```sql
INSERT INTO t_memory (id, memory_id, user_id, agent_id, content, memory_type, status)
VALUES ('mem_001', 'mem_001', 'u_context', 'agent_test', '每天早上一杯美式咖啡', 'preference', 'active');
```

### F-1 按相关性筛选

**说明：** `/search` 默认按 `relevance_score` 降序排列。

**请求：**
```json
POST /api/v1/memory/search
{"query":"咖啡", "user_id":"u_context", "top_k":5}
```

**预期：**
- `code=0`
- `results` 按 `relevance_score` 从高到低排序
- 每条结果包含 `relevance_score` 字段

**代码：** `app/services/memory_store.py:235-240` — sort by score desc

---

### F-2 按类型筛选

**说明：** `/search` 和 `/context` 都支持 `memory_types` 精确筛选。

**请求 — /search：**
```json
POST /api/v1/memory/search
{"query":"偏好", "user_id":"u_context", "memory_types":["preference"], "top_k":5}
```

**预期：** 所有返回结果的 `memory_type` 均为 `preference`

**请求 — /context（新参数）：**
```json
POST /api/v1/memory/context
{"query":"偏好", "user_id":"u_context", "memory_types":["preference"]}
```

**预期：** `fragments` 中所有 `memory_type` 均为 `preference`

**代码：** `app/services/memory_store.py:137` — `Memory.memory_type.in_(memory_types)`

---

### F-3 按状态筛选

**说明：** 新增 `status` 参数，不传默认只查 `active`。

**请求：**
```json
POST /api/v1/memory/search
{"query":"咖啡", "user_id":"u_context", "status":["active"], "top_k":5}
```

**预期：** 返回结果的 `status` 均为 `active`

**边界：**
- 不传 `status` → 等价于 `["active"]`（向下兼容）
- 传 `status=[]` → 等价于不传
- 传 `status=["deleted"]` → 尽力返回（Qdrant 已删向量）

**代码：** `app/services/memory_store.py:105-110` — `_apply_status_filter()`

### L-1 最大返回条数控制

**说明：** `/context` 新增 `top_k` 参数，替代硬编码 20。

**请求：**
```json
POST /api/v1/memory/context
{"query":"咖啡", "user_id":"u_context", "top_k":3}
```

**预期：** `memory_count <= 3`

**代码：** `app/services/memory_store.py:554` — `top_k=top_k`

### L-2 / L-3 文本长度控制 + 长内容裁剪

**说明：** 新增 `max_content_length` 参数，search 默认不截断，context 默认 200。

**请求 — /search（不截断）：**
```json
POST /api/v1/memory/search
{"query":"方案", "user_id":"u_context", "top_k":3}
```

**预期：** `content` 返回完整原文

**请求 — /search（截断）：**
```json
POST /api/v1/memory/search
{"query":"方案", "user_id":"u_context", "max_content_length":50, "top_k":3}
```

**预期：** `content` 长度 ≤ 53（50 + "..."）；原文超过 50 字时末尾为 `...`

**请求 — /context（截断）：**
```json
POST /api/v1/memory/context
{"query":"方案", "user_id":"u_context", "max_content_length":30}
```

**预期：** `formatted_text` 和 `fragments` 中的 `content` 均被截断，长度 ≤ 33

**代码：** `app/services/memory_store.py:96-102` — `_truncate_text()`

### S-1 / S-2 / S-3 结构化结果返回

**说明：** 检索结果必须包含完整的结构化字段。

**请求：**
```json
POST /api/v1/memory/search
{"query":"咖啡", "user_id":"u_context", "top_k":3}
```

**预期（每条结果包含）：**

| 字段 | 说明 | 来源 |
|------|------|------|
| `content` | 记忆内容 | `t_memory.content` |
| `memory_type` | 记忆类型 | `t_memory.memory_type` |
| `relevance_score` | 相关性分数 | Qdrant score |
| `importance` / `confidence` | 重要性 / 置信度 | `t_memory.importance` / `confidence` |
| `agent_id` | 智能体标识 | `t_memory.agent_id`（补齐） |
| `source_type` | 来源类型 | `t_memory.source_type`（补齐） |
| `key_points` | 关键要点 | `t_memory.key_points`（补齐） |
| `scene_id` / `session_id` / `task_id` | 上下文标识 | `t_memory.*_id` |
| `created_at` / `updated_at` | 时间戳 | `t_memory.*_at` |

**代码：** `app/services/memory_store.py:170-186` — 结果构建

### P-1 / P-2 / P-3 Prompt 片段生成

**说明：** `/context` 按类型生成带标记的 Prompt 片段。

**请求：**

**预期：** `formatted_text` 包含以下段落（有数据时）：
- `⭐ 用户偏好`
- `📋 关键事实`

**代码：** `app/services/memory_store.py:480-497` — `_format_grouped()`

### P-4 / P-5 / P-6 上下文组织输出

**说明：** `/context` 支持分组和排序。

### C-1 / C-2 检索日志记录

**说明：** 每次检索自动写入 `t_retrieval_request` 和 `t_retrieval_result`。

**操作：** 执行任意 `/search` 或 `/context` 请求

**验证：**
```sql
-- 查检索请求
SELECT request_id, user_id, agent_id, query_text, filter_conditions, top_k
FROM t_retrieval_request
WHERE user_id = 'u_context'
ORDER BY created_at DESC LIMIT 5;

-- 关联检索结果
SELECT r.request_id, r.memory_id, r.rank, r.relevance_score
FROM t_retrieval_result r
JOIN t_retrieval_request q ON r.request_id = q.request_id
WHERE q.user_id = 'u_context'
ORDER BY r.request_id, r.rank;
```

**预期：** 请求表有记录，结果表有对应的 `memory_id` / `rank` / `relevance_score`

**代码：** `app/services/memory_store.py:31-58` — `_log_retrieval()`

## 执行顺序（Postman / 手动）

```
 1. POST /memory/search (无过滤)          → F-1  基础检索
 2. POST /memory/search (memory_types)    → F-2  类型筛选
 3. POST /memory/search (status)          → F-3  状态筛选
 4. POST /memory/search (max_content_len) → L-2  截断
 5. POST /memory/context (top_k=3)        → L-1  条数控制
 6. POST /memory/context (memory_types)   → F-2  类型筛选
 7. POST /memory/context (max_content_len)→ L-3  内容截断
 8. POST /memory/context (group_by_type)  → P-4/P-5/P-6  分组/排序/格式化
 9. 查 t_retrieval_request 表               → C-1/C-2  日志写入
10. 查 t_api_log 表                         → C-3  耗时记录
11. POST /memory/search (模拟异常)         → E-1/E-2/E-3  错误码
```
