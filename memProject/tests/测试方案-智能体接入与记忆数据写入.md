# 测试方案：智能体接入与记忆数据写入

> Base URL: `http://localhost:8000`
>
> 统一请求格式：`messages` 为数组，每项包含 `role` + `content` 两个字段

## 涉及接口

| 接口 | 路径 | 代码 |
|------|------|------|
| 注册 | `POST /api/v1/agent/register` | `app/api/v1/agent.py:35` |
| 写入 | `POST /api/v1/memory/write` | `app/api/v1/memory.py:125` |
| 异步 | `POST /api/v1/memory/async_write` | `app/api/v1/memory.py:174` |
| 查询 | `GET /api/v1/agent/{id}` | `app/api/v1/agent.py:82` |

---

## A. 智能体接入管理

### A-1 智能体标识 + 场景标识 + 接口密钥

**请求：**
```json
POST /api/v1/agent/register
{"agent_name":"测试智能体A","scene_id":"scene_test"}
```

**预期：** `agent_id=agent_xxx`, `api_key=mem_xxx(68位)`, `scene_id=scene_test`

**验证：** Navicat `t_agent` — `api_key_hash` 是 SHA256，无明文字段

**代码：** `app/api/v1/agent.py:35-79`, `app/core/security.py`

---

## B. 接入参数管理

> 核心验证：参数从 Header/Body 注入到数据库的正确性。`messages` 的 content 只是填充。

### B-1 用户ID — Body 传入

```json
POST /api/v1/memory/write
{"user_id":"test_user_001","messages":[{"role":"user","content":"我叫张三"}]}
```

**验证：** `t_interaction_record.user_id = "test_user_001"`

### B-2 用户ID — Header 优先于 Body

```
Header: X-User-Id: user_from_header
Body:   {"user_id":"user_from_body","messages":[{"role":"user","content":"测试"}]}
```

**验证：** `t_interaction_record.user_id = "user_from_header"`（Header 优先级更高）

**代码：** `app/api/deps.py:122-134`

### B-3 会话ID配置

```
Header: X-Session-Id: sess_abc123
Body:   {"user_id":"u1","messages":[{"role":"user","content":"测试"}]}
```

**验证：** `t_interaction_record.session_id = "sess_abc123"`

**代码：** `app/api/deps.py:151-156`

### B-4 任务ID配置

```
Header: X-Task-Id: task_xyz789
Body:   {"user_id":"u1","session_id":"sess_001","messages":[{"role":"user","content":"测试"}]}
```

**验证：** `t_interaction_record.task_id = "task_xyz789"`

**代码：** `app/api/deps.py:159-164`

## C. 记忆数据写入

### C-1 用户输入 — 名称提取

```json
{"user_id":"u1","messages":[{"role":"user","content":"我叫张三，是后端工程师"}]}
```

**预期：** `results[0].event = "ADD"`, `memory` 包含 "张三"

**代码：** `app/api/v1/memory.py:92-101`（名称正则）

### C-2 用户输入 — 偏好提取

```json
{"user_id":"u1","messages":[{"role":"user","content":"我喜欢简洁的代码风格"}]}
```

**预期：** `event="ADD"`, `memory` 包含 "简洁的代码风格"

**代码：** `app/api/v1/memory.py:104-111`（偏好正则）

### C-3 用户输入 — 排斥提取

```json
{"user_id":"u1","messages":[{"role":"user","content":"我不喜欢过度设计，讨厌冗余代码"}]}
```

**预期：** `event="ADD"`, `memory` 包含 "过度设计"

**代码：** `app/api/v1/memory.py:114-120`（排斥正则）

### C-4 智能体回复写入

```json
{"user_id":"u1","messages":[{"role":"assistant","content":"好的张三，我记住了你的偏好"}]}
```

**验证：** `t_interaction_record.role = "assistant"`，内容完整存入

### C-5 对话轮次

同一 `session_id` 发 3 次：

```json
{"user_id":"u1","session_id":"sess_turn","messages":[{"role":"user","content":"第一轮"}]}
{"user_id":"u1","session_id":"sess_turn","messages":[{"role":"assistant","content":"回复"}]}
{"user_id":"u1","session_id":"sess_turn","messages":[{"role":"user","content":"第二轮"}]}
```

**验证：** 3 条 `session_id` 相同，`recorded_at` 递增

## D. 数据接入校验

### D-1 必填字段 — 空 content

```json
{"user_id":"u1","messages":[{"role":"user","content":""}]}
```

**预期：** `code=-1`, `error_code: INVALID_PARAM`, `message` 含 "content 不能为空"

**代码：** `app/services/validation_service.py`

### D-2 必填字段 — 缺 user_id

```json
{"messages":[{"role":"user","content":"测试"}]}
```

**预期：** 422, 提示 `user_id` 必填

### D-3 字段类型

```json
{"user_id":123,"messages":[{"role":"user","content":"测试"}]}
```

**预期：** 422, `user_id` 类型错误

**代码：** `app/schemas/memory.py` — Pydantic 自动校验

### D-4 缺 messages 字段

```json
{"user_id":"u1","content":"直接传content"}
```

**预期：** 422, `messages: Field required`

---

## E. 数据标准化

### E-1 时间标准化

```json
{"user_id":"u1","messages":[{"role":"user","content":"测试"}],"timestamp":"2026-07-13"}
```

**验证：** `t_interaction_record.recorded_at` 为 ISO 8601 格式

**代码：** `app/services/validation_service.py`

### E-2 标识标准化

```
Header: X-User-Id: USER_Test_001
Body:   {"user_id":"USER_Test_001","messages":[{"role":"user","content":"测试"}]}
```

**验证：** `t_interaction_record.user_id = "user_test_001"`（去空格、小写）

**代码：** `app/api/deps.py:126`, `app/services/validation_service.py`

### E-3 元数据补全

```json
{"user_id":"u1","messages":[{"role":"user","content":"测试"}]}
```

**验证：** 开发模式 `t_interaction_record.scene_id` 自动补 `"scene_dev_default"`

**代码：** `app/api/deps.py:29 — DEFAULT_DEV_SCENE_ID`
